## Why this is library-worthy
A side-by-side spec-comparison product-block: 2–4 product columns anchored at the top by a clean centered product render + price + paired "Buy Now" / "Learn More" CTAs, then a long cascading spec table beneath organized into category sections (Aircraft, Camera, Sensing, etc.) with a left-rail spec-label column. Solves the missing "compare-our-lineup" pattern for dealers who carry both [R-32](../../../products/r-32.md) and [I-19](../../../products/i-19.md) — the existing product-block notes ([alternating-pastel-stripes](./refero-4bcd783f-alternating-pastel-stripes.md), [dark-feature-bands](./refero-eb12db31-dark-feature-bands.md)) handle single-product capability narratives but neither handles "which one should I buy."

## What to take
- 4-column grid total: a left-rail ~20% column for spec labels, and 3 equal-width product columns (~26% each) for spec values. If the dealer carries only 2 products, use 1 spec-label column + 2 product columns and let the pattern compress horizontally.
- Each product column header is a fixed unit at the top: a small dropdown selector with the model name (lets the user swap which model occupies the column), one centered hero-style product render on white/light backdrop (~180–240px tall, soft contact shadow only — no environmental context), price in plain bold text below the image, then a stacked "Buy Now" filled black pill (~36px tall) + "Learn More" text link with chevron. The header unit is sticky-feeling: it reads as a single anchor the user can return to as the spec rows scroll past.
- Spec rows cascade vertically below, grouped under category section headings (~28px sentence-case heading on its own row, full-width, no rule). Within a category, rows are simple: spec label (~13–14px medium-weight in left rail) + value cell per product (~13–14px regular). Values are short — a number, a unit, a short phrase, or a 1–2-line paragraph. No icons, no graphics, no inline product imagery inside the table.
- Add a small "Hide Duplicated Details" toggle in the upper-left of the table: when products share a value across all columns, the row collapses. This is the move that makes a 100-row spec sheet legible — the user opts into density.
- Page chrome is intentionally restrained: white background, charcoal text, single accent color reserved for "Learn More" links and the "Hide Duplicated Details" toggle. The data IS the visual treatment.
- Vertical rhythm: ~32–48px between category sections, ~12–16px between rows within a category. No striping (zebra rows would fight the category-header rhythm); rely on whitespace alone.

## What NOT to take
- Don't import this pattern if the dealer carries only one product — collapses to a generic spec sheet, and a single-product narrative is better served by the [alternating-pastel-stripes](./refero-4bcd783f-alternating-pastel-stripes.md) (light) or [dark-feature-bands](./refero-eb12db31-dark-feature-bands.md) (dark) patterns.
- Don't overload with rows. DJI's sheet runs 100+ rows because they're selling cinematography-tier hardware to professionals who care about gimbal-roll-axis ranges. For a Terraplex dealer, ~15–25 rows organized into 4–5 categories (Airframe, Payload & Spray, Battery & Charging, Operating Envelope, Compliance) is the right density. More than that, the user stops reading.
- Don't blend marketing prose into spec values. A row is one number, one unit, or one short phrase per cell — never a paragraph of feature copy. Marketing prose belongs in a separate product-block above this one (use [alternating-pastel-stripes](./refero-4bcd783f-alternating-pastel-stripes.md) or [dark-feature-bands](./refero-eb12db31-dark-feature-bands.md) as the lead, then this pattern as the closer for buyers ready to compare).
- Don't add a configurable "swap in a different model" dropdown beyond what the dealer carries — the dropdowns in the source allow swapping among the consumer-Mavic family, which only works because DJI has 8+ models. With 2–3 dealer models, fix the columns; don't make them user-configurable.
- The dense engineering-spec aesthetic conflicts directly with [FIELD's](../../../archetypes/field.md) warm-photographic, no-clinical-tables rule and [PATRIOT's](../../../archetypes/patriot.md) heritage register. The cold typographic table reads as catalog, not story — wrong tonal register for both archetypes. For warm archetypes, see Adaptation notes below for a softer fallback (or skip the pattern).
- Don't put the price next to a "Buy Now" pill if the dealer's pricing strategy is MSRP-only or kit-tier-redacted — see [content/pricing-policy.md](../../../content/pricing-policy.md). The pattern still works without prices; the CTAs become "Request Quote" / "View Spec Sheet" instead.

## Adaptation notes (cross-archetype)
- TACTICAL: native fit. Tighten geometry — sharper rectangle "Buy Now" buttons (~6px corner radius instead of pill), precision-sans for spec labels and category headings, monochrome black/white only with a single steel-cyan accent on the "Learn More" links. Add a thin 1px corner-bracket detail to the product-column headers (consistent with the archetype's bracket motif). Hardware photographed on seamless black studio paper. See [TACTICAL archetype](../../../archetypes/tactical.md). Pairs naturally with the [hardware-lineup-dark-hero](../../hero/research/refero-0945a36c-hardware-lineup-dark-hero.md) above this product-block.
- CLEAN: native fit. White background, neutral grey grid lines (optional, very subtle), single brand-navy accent on "Learn More" links and the "RECOMMENDED" pill (if used — only on one product, no scaling). See [CLEAN archetype](../../../archetypes/clean.md). Pairs with the [centered-screenshot-trio hero](../../hero/research/refero-f619e700-centered-screenshot-trio.md) above.
- TERRAIN: works as a "light band" insert in the alternating rhythm. Add a thin topographic-line divider above each category section heading (replaces the rule the source omits). Spec labels and category headings in clay or sage accent only; values stay charcoal. See [TERRAIN archetype](../../../archetypes/terrain.md).
- FORGE: invert to a dark surface — charcoal background (#1A1A1A), white spec values, ember-orange "Buy Now" pills (only on the product the dealer wants to anchor; the others stay white-on-dark for contrast). Keep the product renders on their own light tile inside each column header (a ~16px-padded white rounded card around each render) so they read clearly against the dark page — the renders need contrast that a dark seamless studio shot can't give them at this scale. Risk to manage: the dense spec table feels heavy on dark; if it does, lift the spec block onto a slightly-lighter neutral panel (~#222) while keeping the page chrome dark. See [FORGE archetype](../../../archetypes/forge.md).
- FIELD / PATRIOT: skip the literal pattern. If a warm-archetype dealer genuinely needs a side-by-side, rebuild as an editorial-style two-column spread rather than a spec table: two product photographs side-by-side at large scale, each below a short paragraph describing the model's role in plain English ("Built for orchard rows under low canopy" vs "Built for open prairie at scale"), with values appearing inline in the prose ("carries 80 lbs", "flies for 30 minutes") rather than in a table. The cold-table register conflicts with both archetypes' heritage / warm-photographic rules — the table IS the affordance, and softening it past a certain point just produces a worse comparison.

## Source brief
DJI's [/products/comparison-consumer-drones](https://www.dji.com/products/comparison-consumer-drones) page — the consumer-Mavic spec-compare landing for Mavic 3 Pro / Mavic 3 / Mavic 3 Classic. Most directly transferable reference in the library because it's the same domain (drones) Terraplex dealers operate in, and the same buying decision (which model fits my use case) the dealer's customers face when choosing between [R-32](../../../products/r-32.md) and [I-19](../../../products/i-19.md). The library-worthy move is the structural cadence (sticky product-column headers with render + price + paired CTAs → cascading category-grouped spec rows → "Hide Duplicated Details" density toggle), reusable for any [TACTICAL](../../../archetypes/tactical.md) / [CLEAN](../../../archetypes/clean.md) / [TERRAIN](../../../archetypes/terrain.md) / [FORGE](../../../archetypes/forge.md) dealer site without dragging DJI's specific row count or content along.

Sibling product-block notes: [alternating-pastel-stripes](./refero-4bcd783f-alternating-pastel-stripes.md) for the light-archetype single-product capability narrative; [dark-feature-bands](./refero-eb12db31-dark-feature-bands.md) for the dark-archetype single-product capability narrative. This spec-comparison pattern is the right pick when the dealer needs to surface "which model" as a discrete buying decision; the other two are right when the page is leading a customer through a single product's story. Often used as the closer below one of those — narrative first, comparison second.
