## Why this is library-worthy
A hero pattern where the **actual product UI is embedded in the hero band**, not a static screenshot — full-bleed cinematic photograph as the hero backdrop with a working live UI panel floating over it (vehicle picker, route planner, mission-plan map, live spec readouts updating in real time). Headline + subhead overlay sits in the upper-left ~40% above the photograph; the embedded UI panel anchors lower-center or lower-right, with a "Run simulation" or "Try a full demo" CTA inside the panel. The hero is the demo. Different from existing hero patterns: the [centered-screenshot-trio](./refero-f619e700-centered-screenshot-trio.md) shows STATIC product UI surfaces; the [form-in-hero overlay](./refero-7d15989d-form-in-hero-overlay.md) collapses lead capture into the hero (form, not demo); the [hardware-lineup dark hero](./refero-0945a36c-hardware-lineup-dark-hero.md) shows hardware on a sweep. This pattern shows the **product running** — the user can interact with a slimmed-down version of the actual planning / monitoring / mission UI before scrolling. Right pick when the dealer's product has a real interactive surface (mission planner, fleet console, compliance dashboard) and the buying-decision hesitation is "show me what this actually does."

## What to take
- Two-layer hero band (~70vh, contained — not full viewport): bottom layer is a full-bleed cinematic photograph (top-down aerial of equipment in operating context — fleet of trucks at a charging station, drones over crop rows, equipment laid out at a maintenance facility). Photograph is desaturated ~10–15% so the foreground UI panel reads as primary. Top layer is the headline overlay in the upper-left ~40% and the live UI panel floating in the lower-center ~60%.
- Headline overlay: 2–3 line bold sentence-case headline in white (~36–48px), single subhead line in muted gray (~16px), no CTA — the CTA lives inside the embedded UI panel. The headline names the operational benefit ("Routing for EV fleet management" → for a dealer "Mission planning for variable-rate spray applications" / "Compliance logging for Part 137 operations").
- Embedded UI panel: a working slimmed-down product surface, ~700–900px wide, anchored at the bottom-center or bottom-right of the hero band. Panel chrome is dark (~#1A1A1A) with a thin hairline border (~#2A2A2A) — pulls from the dealer's actual product chrome so the page reads as "the product, not a marketing visualization."
  1. **Top of panel**: a 2-column input row — "Find the best EV / mission / aircraft for your operations" type heading on the left, 2–3 input fields on the right (Origin / Destination / Vehicle dropdown for EVs; Crop / Field-block / Aircraft for spray; Mission-type / Date / Operator for compliance). Inputs are pre-populated with sensible defaults so the panel doesn't read empty.
  2. **Mid panel**: a list of selectable rows below the inputs — vehicle / aircraft / mission options with key specs (range, payload, charge time / flight time / fuel burn). Each row is a thin selectable cell with a tiny thumbnail + name + 2–3 spec callouts. Selected row highlights with a thin accent edge.
  3. **Right panel**: a live-updating data block — a small 4-up metric grid (Distance / Battery / Energy / Consumption — for a dealer Acres / Payload / Flight-time / Coverage), each metric a bold number + unit + small label below. The metrics update when the user changes inputs or selects a different row. Includes a small chart sparkline below the metrics.
  4. **Map / canvas region**: a small map embed (or mission-plan overlay, or compliance-region map) at the bottom of the panel showing the planned route / mission boundary / station network — pin markers representing real waypoints. The map is interactive at minimum-viable level (pan / zoom; clicking pins shows a tooltip).
  5. **Bottom CTA bar**: a single full-width filled pill CTA inside the panel ("Run a full demo" → for a dealer "Get a tailored mission plan" / "Try the compliance log builder"). One CTA, one conversion action — the rest of the panel is the affordance.
- Below the hero band, the page proceeds with normal product / about / pricing sections — the embedded UI in the hero already demonstrated the affordance, so the body sections can stay narrative rather than re-demonstrating.

## What NOT to take
- Don't import this pattern if the dealer doesn't have a real working product surface to embed. A static screenshot of the product UI dressed up to look interactive is the worst version of the pattern — the user clicks, nothing happens, and the page's credibility collapses. If the dealer's only artifact is a static UI screenshot, use the [centered-screenshot-trio hero](./refero-f619e700-centered-screenshot-trio.md) instead — that pattern is honest about being static.
- Don't make the embedded UI a marketing facsimile of the product. The panel must use the actual product's chrome — inputs, fields, typography, colors, button shapes — so the user understands "this is what the tool actually looks like." A pretty marketing visualization that doesn't match the product is misleading and dilutes trust.
- Don't import Chargetrip's literal EV / charging-station copy or the cream-on-dark color treatment — those are EV-specific. The transferable move is "live product UI as the hero affordance" applied to whatever the dealer's actual product surface is. For Terraplex dealers this is mission planning ([R-32](../../../products/r-32.md) / [I-19](../../../products/i-19.md) flight planner), spray-pattern simulation, fleet status dashboard, or compliance log builder ([Part 137](../../../content/compliance-program.md) record builder).
- Don't put more than ONE CTA inside the embedded panel. The pattern's conversion path is "interact with the demo, then click the panel CTA." A second CTA ("Or contact sales →") dilutes the affordance and signals to the user that the demo is decoration.
- Don't pair this with the [form-in-hero overlay](./refero-7d15989d-form-in-hero-overlay.md) on the same page — both occupy the "interactive hero" slot. Pick one: form-in-hero collapses lead capture into the hero (gets the email); demo-in-hero collapses product trial into the hero (gets the engagement). Different conversion paths, different page jobs.
- Don't combine the embedded UI with a video autoplay loop in the background photograph. The UI panel is the interactive layer; an autoplay video on the photograph below competes for attention. Use a static cinematic photograph; let the panel carry the motion.
- The cream-default backdrop and warm-photographic register of FIELD and PATRIOT conflict with this pattern's dark cinematic aesthetic. Forcing a cream-tinted version of the photograph and a light-chrome embedded UI loses the panel-floats-over-cinematic-photo affordance — there's nothing for the panel to lift off of. For warm archetypes, use the [hands-on craft hero](./refero-be842c1e-hands-on-craft-hero.md) (warm-interior real-person mid-use) instead.

## Adaptation notes (cross-archetype)
- TACTICAL: native fit. Tighten geometry — sharper rectangle panel corners (~6px corner radius), precision-sans for panel labels and field placeholders, monochrome white-on-near-black with a single steel-cyan accent on the selected-row highlight, the metric numbers, and the bottom CTA. Add thin 1px corner-bracket details to the panel corners (consistent with the archetype's bracket motif). The map embed uses a dark Mapbox style with cyan pin clusters — reads as a precision tracking display. See [TACTICAL archetype](../../../archetypes/tactical.md). Pairs with the [spec-comparison-grid product-block](../../product-block/research/refero-713b66c0-spec-comparison-grid.md) below the hero.
- CLEAN: lighten — the cinematic backdrop photograph stays full-bleed but the embedded UI panel becomes white (~#FAFAFA) with neutral-gray hairline borders, brand-navy accent on selected-row highlight + metric numbers + bottom CTA. The contrast moves from "dark panel on dark photo" to "white panel on dark photo" — same affordance, lighter aesthetic. See [CLEAN archetype](../../../archetypes/clean.md). Pairs with the [centered-screenshot-trio hero](./refero-f619e700-centered-screenshot-trio.md) on a sibling page (this pattern for the "show me what it does" page; centered-screenshot-trio for the "show me the breadth of what it is" page).
- TERRAIN: the embedded UI panel reads as the page's main "interactive band" — add a thin topographic divider on the panel's bottom edge that traces back into the photograph below. Selected-row accent in clay or sage. The map embed uses a custom Mapbox style with contour lines visible at higher zoom (TERRAIN's signature topographic motif). Below the hero, the page alternates with light bands carrying narrative copy. See [TERRAIN archetype](../../../archetypes/terrain.md). Sibling: [topographic layered-bg hero](./refero-ae84b62a-topographic-layered-bg.md) on adjacent pages where no real product surface is available.
- FORGE: native fit. The cinematic backdrop photograph is shot at dusk or with golden-rim lighting (forge-glow on the closest equipment). Embedded UI panel in near-black (~#0E0E0E) with white text, ember-orange accent on selected-row highlight + metric numbers + bottom CTA. Optional ember-warm rim along the panel's right edge — gives the panel a forge-glow feel where it meets the photograph. See [FORGE archetype](../../../archetypes/forge.md). Pairs with the [hardware-lineup dark hero](./refero-0945a36c-hardware-lineup-dark-hero.md) on a sibling page (hardware-lineup introduces the equipment family; this pattern shows it operating).
- FIELD / PATRIOT: skip — the dark cinematic backdrop and dense embedded UI conflict with both archetypes' cream-default and warm-photographic rules. The interactive register reads as corporate-tech, not heritage. For warm archetypes that want to demonstrate product capability, use the [centered-screenshot-trio hero](./refero-f619e700-centered-screenshot-trio.md) (3 static screens) or pair the [hands-on craft hero](./refero-be842c1e-hands-on-craft-hero.md) with a body-of-page interactive section (the [maintenance-program-recommender services pattern](../../services/research/refero-d928f889-maintenance-program-recommender.md) handles the interactive moment in a warm-archetype-friendly way).

## Source brief
Chargetrip's [/fleets](https://www.chargetrip.com/fleets) page — an EV-fleet routing-and-charging product landing where the hero band embeds a working route-planner UI (vehicle picker, route inputs, live energy/distance/consumption metrics, interactive charging-station map). The library-worthy move is the structural cadence (cinematic full-bleed photograph backdrop → headline overlay upper-left → working live UI panel anchored bottom-center with inputs + selectable list + live metrics + interactive map + single in-panel CTA), reusable for a Terraplex dealer who has built or is building a real interactive product surface — mission planning, spray-pattern simulation, fleet console, [Part 137 compliance log builder](../../../content/compliance-program.md). The pattern's whole credibility move is "the demo IS the page" — no separate "request a demo" page needed when the demo is the first thing the user sees.
