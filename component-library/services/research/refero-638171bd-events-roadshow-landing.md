## Why this is library-worthy
A services pattern for **events / roadshows / field days** — the dealer's calendar of in-person service touchpoints (Part 107 prep cohort sessions, field-day demos, county fair appearances, Sam Welton compliance clinics, training events). The library has 14 services patterns covering capability menus, tier comparisons, audience anchoring, recommenders, process flows, and case studies — none cover "where we'll be next / where we've been" as the page's job. This pattern fills that gap with three coherent affordances: a 5–6-cell asymmetric photo collage that proves "we actually show up at things," an explicit `Upcoming` vs `Past Events` split with a graceful empty-state for the upcoming list (Notify-me CTA), and a flat scannable past-events table (Event / Date / Location / Type) that demonstrates breadth-of-presence retroactively rather than via testimonial chrome. Different from the existing services patterns: [photo-mosaic](./refero-2fdf9ace-photo-mosaic-services.md) shows breadth-of-capability in static imagery; this pattern shows breadth-of-time-and-place via dated table rows.

## What to take
- Open with a contained dark or light hero band (~30–40vh — deliberately small): centered serif headline ("Events" / "Field Days" / "Where we'll be next") + 1–2-line subtitle ("Explore what's coming soon and catch up on past events.") + single rounded-pill primary CTA ("See upcoming events" / for a dealer "View this season's schedule"). The hero is intentionally narrow vertically — it's a label, not a marketing pitch. The page's body (collage + table) carries the credibility.
- **Photo collage proof tile** — directly under the hero, a 5–6-cell asymmetric photo grid showing real moments from past events: booth shots, attendee groups, signage, presentations, hands-on demos. Asymmetric tile sizes (one large dominant cell + 4–5 smaller ones) so the eye reads the grid as documentation rather than a uniform card row. Photos must be unedited reportage register — no studio lighting, no posed marketing shots; the credibility move is "this is what we actually look like at events." A small "Notify me" pill CTA centered below the collage with explanatory line ("To be notified about future events, subscribe to our [calendar / newsletter / Luma].")
- **`Upcoming Events` section** — centered section title + light subtitle ("Join us at future events"). Card list of upcoming events (1 per row): event name (bold) + date + location + small category pill ("Conference / Field Day / Cohort Session / Demo"). When there are no upcoming events, the section MUST show a graceful empty state — never hide the section. Two acceptable empty-state treatments:
  - "There are no upcoming events at this time. In the meantime, please explore [replays / past events] found below." (light register — Teachable approach)
  - A single "Notify me" CTA repeated with a 1-line explanation ("To be notified about future events, subscribe to our [calendar].") (dark register — Resend approach)
- **`Past Events` section** — the pattern's distinguishing affordance. Choose ONE of two structural variants based on dealer brief:
  - **Table variant (dense / breadth-led):** 4-column flat table — `Event` / `Date` / `Location` / `Type`. 12–24 rows visible by default, hairline rules between rows, no card chrome. The `Type` column holds a small pill tag ("Conference / Meetup / Field Day / Cohort / Demo / Workshop / Online"). The dense table form does the credibility work because it's *scannable* — the visitor sees 20 city names and 24 dates and reads "they actually go places" before reading any single row. Ideal for dealers with 10+ past events.
  - **Card variant (visual / replay-led):** 3-up card grid (~6 visible by default), each card with thumbnail photo + title + 2-line description + "WATCH REPLAY" or "READ RECAP" link below. "Load more" pagination CTA centered below the grid. Ideal for dealers with rich event content (recordings, recaps, photo galleries) and < 12 past events.
- Keep the rest of the page minimal — the page closes with the standard footer. No closer CTA band, no testimonial band, no FAQ. The events ARE the page.

## What NOT to take
- Don't use this pattern if the dealer's events are sparse (< 4 past events AND no upcoming). The empty-state plus an underpopulated past-events table reads as "we don't really do events" — the opposite of the credibility move. For dealers without an active event cadence, skip the events page entirely or fold a single "Upcoming demo" line into the [contact horizontal-row directory](../../contact/research/refero-c29ba321-horizontal-row-directory.md).
- Don't fabricate the photo collage. The collage's credibility is "we actually show up at events" — staged stock photos defeat the entire move. If the dealer has < 5 real event photos, skip the collage and go straight to the past-events table; an unstaged 4-photo grid undersells less than a staged 6-photo grid oversells.
- Don't mix variants (table + cards on the same page). Pick one register based on what the dealer actually has. Mixing reads as indecision and breaks the page rhythm.
- Don't let the table row count exceed ~30 visible-by-default. Beyond 30, the table loses scannability and turns into a wall of text. Add pagination ("Load more") or a year-filter strip ("2026 / 2025 / 2024") instead. The Resend version shows 17 rows then footer — that's the sweet-spot density.
- Don't add a CTA to every past-event row. The past-events list is documentation, not conversion. Adding "Watch replay" or "View recap" links to every row only makes sense in the card variant where the recap content actually exists. In the table variant, rows are inert by design — the breadth itself is the message.
- Don't use generic event categories ("Other / Misc / Various"). The `Type` pill must do meaningful classification work — "Conference / Field Day / Demo / Workshop / Online / Cohort Session" — otherwise the column is decorative chrome. If the dealer's events all collapse to one type, drop the `Type` column entirely (3-column table reads cleanly as Event / Date / Location).
- Don't skip the upcoming-events empty state. Hiding the section when there's nothing scheduled signals "we ran some events once and stopped" — worse than the graceful "Notify me" treatment. The empty state with a notify-CTA promises continuity even between events.
- Don't import Resend's literal serif-on-near-black palette or Teachable's pastel-card register — both are source-specific. The transferable move is "small dark/light hero + asymmetric photo collage + explicit upcoming/past split + flat dense scannable table OR replay card grid." Colors come from the archetype.

## Adaptation notes (cross-archetype)
- FIELD: cream page background (~#F4EFE7), warm-charcoal serif headline ("Field Days" works native here), photo collage of golden-hour booth/attendee shots from county fairs and farm-equipment expos. Card variant fits FIELD better than table — the warm photographic register asks for visual cards, not data rows. `Type` pills in deep amber. See [FIELD archetype](../../../archetypes/field.md). Pairs with the [hands-on craft hero](../../hero/research/refero-be842c1e-hands-on-craft-hero.md) on a parent landing page.
- CLEAN: white page background, brand-charcoal serif headline, photo collage with consistent soft-shadow framing on each cell. Either variant works — table variant for breadth, card variant for content depth. `Type` pills in brand-navy with white text. See [CLEAN archetype](../../../archetypes/clean.md).
- TERRAIN: alternating cream/dark band rhythm carried through the page — hero on cream, collage band on dark, upcoming on cream, past-events on dark. Add a thin topographic divider between Upcoming and Past Events sections. Table variant works native here — the dense data rows pair well with the contour-line minimalism. `Type` pills in clay or sage. See [TERRAIN archetype](../../../archetypes/terrain.md).
- PATRIOT: cream page background, deep-navy serif headline, photo collage of American-farmland event shots (real farmers, real county fairs, real cooperative-extension days). `Type` pills in barn-red with cream text. Card variant pairs with the [manifesto-creed hero](../../hero/research/refero-b86571f2-manifesto-creed-hero.md). See [PATRIOT archetype](../../../archetypes/patriot.md).
- FORGE: native fit for the dark Resend register. Charcoal page background (~#1A1A1A), white serif "Events" headline, photo collage of equipment-demo dusk shots (operators with rim-lit equipment at field-day events). Table variant works well in dark — rows separated by hairline ember-glow rules. `Type` pills in ember-orange-on-near-black. See [FORGE archetype](../../../archetypes/forge.md). Pairs with the [hardware-lineup dark hero](../../hero/research/refero-0945a36c-hardware-lineup-dark-hero.md) above.
- TACTICAL: tighten geometry — sharper rectangular tiles in the photo collage (no rounded corners), precision-sans for the headline (consider dropping the serif for an all-caps grotesque "EVENTS"), monochrome white-on-near-black with steel-cyan only on `Type` pills and table column headers. Add 1px corner-bracket details to the collage cells. Table variant strongly preferred — TACTICAL anti-patterns punish loose card layouts. See [TACTICAL archetype](../../../archetypes/tactical.md).

## Source brief
[Resend's `/events` page](https://resend.com/events) — a developer-focused email-API marketing landing. Resend uses the page as social-proof-by-presence: a 6-cell collage of past conference booths and demo days, a near-empty `Upcoming Events` slot with a Notify-me CTA, and a 17-row flat past-events table covering 8 months of conferences, meetups, hackathons, and online events across 6 cities and 2 countries. The library-worthy move is the structural cadence (small label-shaped hero → asymmetric photo collage → notify-me CTA → upcoming/past split → flat scannable table) — directly reusable for a Terraplex dealer's events / roadshows / field-days page. The Teachable variant (`/events`, light register, replay-card grid instead of table) demonstrates the same cadence in a card-led register, so the pattern travels across both dense and visual treatments.
