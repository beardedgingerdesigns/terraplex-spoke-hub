## Why this is library-worthy
A services pattern for the dealer's **training / course / curriculum catalog** — a filterable list of self-paced learning products (Part 107 prep, advanced operator training, fleet manager certification, drone-specific maintenance courses) where each course has its own level + topic + duration metadata. The library has 16 services patterns covering capability menus, tier comparisons, audience anchoring, recommenders, process flows, case studies, calculators, ROI projections, single-plan explainers, and the in-person events landing — none cover "browse our self-paced course library and filter by skill level / topic" as the page's job. This pattern fills that gap with three coherent affordances: a saturated brand-color hero that labels the whole catalog (deliberately small — it's a header, not a marketing pitch), a left filter rail with two collapsible filter groups (Level + Topic) showing live counts in parentheses so the visitor can see what's filterable at a glance, and a vertical stack of horizontal course rows where each row's left thumbnail tile is color-coded by topic (so the catalog reads as a visually-categorized library even before the filter is engaged).

## What to take
- Open with a saturated brand-color hero band (~25–30vh — deliberately small): centered display headline ("All courses" / "Training library" / "Operator certification courses") + 1-line subtitle ("Browse the full catalog of courses offered."). No CTA pill in the band — the catalog itself is the affordance. Hero is intentionally narrow vertically — it's a label, not a pitch.
- **Left filter rail (sticky on scroll, ~200px wide).** Two collapsible filter groups stacked top-to-bottom:
  - `Level` — checkbox list with three options: `Beginner` / `Intermediate` / `Advanced`. Each option shows a course count in muted parentheses next to the label (e.g., `Beginner(12)`). The count updates live as other filters apply.
  - `Topic` — checkbox list with 3–6 dealer-specific topic options (e.g., for a Terraplex dealer: `Part 107 prep / Spray operations / Maintenance / Fleet management / Compliance`). Same parenthetical-count treatment. Options with `(0)` count remain visible but disabled-looking — the zero count itself is information ("we don't have a course on that topic yet").
  - Top of the rail: small `Collapse All` and `Reset` text-links (no chrome, just blue/accent text). The Collapse All collapses both filter groups; Reset clears all checkboxes. Both are required affordances — a 15+ row catalog with no escape hatch from a bad filter combination is unusable.
- **Course-row stack (right of filter rail, full remaining width).** Each course occupies one row, ~110–130px tall, separated by a thin hairline rule. Row anatomy left-to-right:
  - **Color-coded thumbnail tile** (~180×110px) on the left. The tile background color is the visual marker for the course's topic — green tile for one topic family, navy for another, beige for another, purple, orange. The tile holds the course title in 2-line bold sans-serif at the top-left + a small dealer wordmark at the bottom-left. The course's color category tells the visitor at a glance which "row" of the curriculum it belongs to without engaging the filter.
  - **Module/step icon strip** (3–4 small monochrome icons stacked vertically OR clustered in a 2×2 grid) directly right of the thumbnail. These icons preview the course's modules (lesson types: video, reading, quiz, lab, certificate). Decorative chrome — they signal "this is structured curriculum" without listing the modules in text. Optional but improves scannability.
  - **Course metadata block** (right ~60% of the row): course title (bold sans-serif, ~18–20px) + 1-line description (regular weight, ~14px, single line — truncate with ellipsis if it overflows) + duration tag below in muted gray (`13 min` / `2 hours` / `4 weeks`). The duration tag is the second non-negotiable metadata element after title — it sets up the customer's "how much of my time will this take?" judgment before they click in.
- **No CTAs at the row level.** Each row is the click target itself (entire row → course detail page). Adding a "Start course" button to every row clutters the catalog and breaks the visual scan.
- **Footer is the standard site footer.** No closer CTA band, no testimonial band, no FAQ. The catalog IS the page.

## What NOT to take
- Don't use this pattern if the dealer has fewer than ~6 distinct training courses. The filter rail's value is the count math (`Beginner(12)` / `Advanced(4)`) — with 4 courses total, the filter is decorative chrome and the catalog should collapse to a flat 4-row stack with no filter rail (or just the [horizontal row directory](../../contact/research/refero-c29ba321-horizontal-row-directory.md) as a contact-style pivot to a "Contact us about training" CTA). The pattern's credibility move is breadth-of-curriculum.
- Don't fabricate the topic-color mapping. The thumbnail tiles work because each color reliably maps to a topic family (e.g., all `Part 107 prep` courses are green, all `Spray operations` are amber, all `Fleet management` are navy). Random tile colors break the visual-categorization affordance and reduce the tiles to decoration. Pick the topic-color mapping once at design time and apply it consistently across the catalog.
- Don't combine this pattern with the [in-person events / roadshow landing](./refero-638171bd-events-roadshow-landing.md) on the same page. Self-paced courses (this pattern) and in-person events (the roadshow pattern) answer different visitor jobs ("learn at my own pace" vs "where will the dealer be next?") and mixing them on a single page degrades both. Link from one to the other in the page's footer or in a top-of-page nav strip — but keep the catalogs structurally separate.
- Don't add per-row pricing pills. If the dealer's courses have differentiated pricing tiers, that decision belongs on the course-detail page or in a separate [three-tier inverted-middle](./refero-15fcbffc-three-tier-inverted-middle.md) tier-comparison block — not as a fifth column in the catalog row, which would balloon row height and fight the visual scan. The catalog answers "what is offered"; pricing answers "what does it cost"; let those be two separate jobs.
- Don't omit the course count in the filter labels. `Beginner` alone reads as a list of options; `Beginner(12)` reads as catalog metadata. The count is what makes the filter rail feel like a real index rather than a Wix-template chrome decoration. If the dealer has only 1–2 courses per filter cell, the filter rail itself isn't justified — fall back to a flat catalog without filtering.
- Don't let row count exceed ~25 visible-by-default. Past 25 rows the catalog loses scannability and turns into a wall of horizontal cards. Add pagination (`Load more` link below the stack) or strengthen the topic-filter granularity to slice the catalog into smaller working sets. The Zapier source shows ~17 rows then footer — that's the sweet-spot density.
- Don't import the source's all-uppercase `BROWSE THE FULL CATALOG` copy register or its specific orange-on-peach hero palette. Those are source-specific. The transferable move is "small saturated hero label + left filter rail with parenthetical counts + horizontal course-row stack with topic-color thumbnail tiles + duration tag." Colors and copy register come from the archetype.

Filter-rail labels in warm-charcoal; counts in muted clay. See [FIELD archetype](../../../archetypes/field.md). - CLEAN: white page background, brand-navy hero band, charcoal sans-serif headline. Topic-color mapping uses brand-navy + 3–4 hue rotations of the brand accent (each in low saturation so the catalog reads calm). Filter-rail labels in charcoal; counts in 60% gray. See [CLEAN archetype](../../../archetypes/clean.md). Pairs with the [type-only stats hero](../../hero/research/refero-14f39897-type-only-stats-hero.md) when the dealer's curriculum credibility is "X courses, Y graduates."
- TERRAIN: cream/dark alternating page rhythm — hero on dark with a warm-amber headline, filter rail and catalog on cream. Topic-color mapping uses contour-line palette (clay, sage, slate, warm-charcoal). Add a thin topographic divider between the filter rail and the catalog stack. See [TERRAIN archetype](../../../archetypes/terrain.md).
- PATRIOT: cream page background, deep-navy hero band, ivory display-serif headline ("Operator School" / "Compliance Academy" reads native). Topic-color mapping uses barn-red, deep-navy, prairie-amber, charcoal. Filter-rail labels in deep-navy; counts in muted barn-red. See [PATRIOT archetype](../../../archetypes/patriot.md). Pairs with the [manifesto-creed hero](../../hero/research/refero-b86571f2-manifesto-creed-hero.md) above when the curriculum is framed as a creed-aligned credentialing ladder.
- FORGE: native fit for a dark register. Charcoal page background (~#1A1A1A), saturated ember-orange hero band, near-white sans-serif headline. Topic-color mapping uses ember-orange, steel-blue, bronze, deep-violet — all saturated against the dark page. Filter rail on near-black (~#0F0F0F) with white labels; counts in dim ember. Hairline rules between rows in a 1px ember-glow accent. See [FORGE archetype](../../../archetypes/forge.md). Pairs with the [emblem-program hero](../../hero/research/refero-c1c2abcf-emblem-program-hero.md) when the catalog is positioned as the curriculum behind a credentialing program.
- TACTICAL: tighten geometry — sharper rectangular thumbnails (no rounded corners), precision grotesque sans for the headline (consider `BROWSE COURSES` all-caps), monochrome white-on-near-black with steel-cyan only on filter checkboxes and duration tags. Add 1px corner-bracket details to each thumbnail tile. Topic-color mapping uses 4 hues max — steel-cyan, ember-amber, bone-white, dim-graphite. See [TACTICAL archetype](../../../archetypes/tactical.md). Pairs with the [3D-object dark hero](../../hero/research/refero-ea86d249-3d-object-dark-hero.md) on a parent landing page.

## Source brief
[Zapier Learn's `/page/courses` catalog](https://learn.zapier.com/page/courses) — a free training catalog for a SaaS automation platform. Zapier surfaces ~17 self-paced courses across two filter groups (Level: Beginner / Intermediate / Advanced; Topic: Lead management / Sales & rev ops / Security) with parenthetical counts on each filter option. Each course row has a color-coded thumbnail tile (one of ~5 topic-family colors), a module-icon strip, a bolded title, a 1-line description, and a duration tag (`13 min` / `35 min`). The library-worthy move is the structural cadence (small saturated hero label → left filter rail with live counts → horizontal course-row stack with topic-color thumbnails + duration tags) — directly reusable for a Terraplex dealer's training catalog page when the dealer has 6+ self-paced courses across 2+ skill levels.

Sibling services notes (this pattern's place in the 17-pattern services library):
- For "in-person events / roadshows / field-days" → [events-roadshow-landing](./refero-638171bd-events-roadshow-landing.md). Self-paced training (this note) and in-person events are different visitor jobs.
- For "I don't know which course / program is right for me" → [maintenance-program-recommender](./refero-d928f889-maintenance-program-recommender.md). The recommender is a wizard; this catalog is a browseable library. They compose: a recommender at the top of the curriculum landing → catalog below for self-directed browsing.
- For "tell me what each tier of training costs" → [three-tier-inverted-middle](./refero-15fcbffc-three-tier-inverted-middle.md). Pricing tiers are different from a catalog of distinct courses — the tier-card pattern compares plans, this pattern lists products.
- For "we have one named training program (Part 107 prep)" → [protection-plan-explainer](./refero-a52e1484-protection-plan-explainer.md) (single-plan explainer pattern, applied to a single named training program rather than a service contract). When the dealer has only 1 training product, that pattern fits better than this catalog.
- For "show me the steps of enrollment" → [numbered-process-shopper](./refero-d030e2b2-numbered-process-shopper.md). Process and catalog compose — process at the bottom of the curriculum landing, catalog above.

Pick guide: this filtered-course-catalog pattern when the dealer has 6+ self-paced training courses across 2+ skill levels AND wants visitors to browse-and-filter rather than be wizard-routed. Otherwise pick the recommender (wizard-routed for "help me choose") or the single-plan explainer (one named training product).
