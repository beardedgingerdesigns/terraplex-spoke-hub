## Why this is library-worthy
A contact pattern where the page IS a scheduling widget — left column is a 2-line "Book A Demo" heading + 1-line supporting copy; right column is a single elevated card stack with three sequential layers: rep avatar+name+title at top, intake form in the middle, calendar grid + timeslot picker on the right. The conversation pre-frame (a real rep's name and face) reframes the contact action from "submit a form so we get back to you" to "you're booking a slot with [Jeremy]." Different from existing contact patterns: the [two-card decision pivot](./refero-376ca585-two-card-decision-pivot.md) routes intent before the form; the [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md) previews conversation topics; the [dealer-locator map](./refero-71707203-dealer-locator-map.md) routes by location. This pattern routes by **time** — the dealer has booked timeslots, and the customer self-serves into one. Right pick when the dealer offers scheduled demo flights, [Part 107 prep consultations](../../../content/compliance-program.md), or fleet check-ins where the value is "guaranteed time with someone, soon."

## What to take
- Two-column band: left ~35% is heading + supporting copy + small site nav strip ("Pricing / Changelog / Book A Demo / Login / Get Started"). Right ~65% is the calendar widget — implemented as a single elevated card with a soft drop shadow (~20px blur, ~8% opacity) and ~16px corner radius.
- Inside the card, three sequential layers (top to bottom):
  1. **Rep card pre-frame** — small avatar (~48px circle) on the left, rep name (~16px bold) + role/title (~13px muted gray) below the name, then a short context line ("See how we focus on…" or for a dealer "Walks you through R-32 fleet planning"). The rep is real — a named person, not "the team." This is the credibility move; without a real name + face, the pattern collapses into a generic Calendly embed.
  2. **Intake form** — 5–7 short fields stacked vertically: Email, Company, Company Website, Team Size dropdown, 1–2 contextual radio rows ("Do you work with clients? Yes/No"), 1 short text area ("What [domain] tools do you currently use?"), and a plan-selector radio group ("Basic / Extended (recommended) / Pro / I'm not sure yet"). Labels sit above the fields; required fields use a small red asterisk. The form is short — under one screen on desktop — so the calendar stays visible.
  3. **Calendar grid + timeslot picker** — month-and-year header at top, day-of-week column headers, then a date grid where un-bookable days are dimmed and bookable days are crisp. Below the date grid, a vertical scrollable list of timeslots for the selected date (each ~40px tall pill, white with 1px border). Selected timeslot turns brand-accent (blue / amber / navy / cyan per archetype). A small "Scroll for more slots" anchor at the bottom of the timeslot list.
- Submit CTA is a single filled pill ("Schedule") inside the card, anchored bottom-right of the form layer — NOT a separate band below the card. The whole conversion happens inside the card.
- Bottom-right of the page corner: a small chat-bubble support widget (~48px) for users who need to ask a question instead of book — the pattern's escape hatch.
- Page background is plain off-white or pale cream; the card carries all the visual weight.

## What NOT to take
- Don't import this pattern without a real rep — the pattern's whole credibility move is "you're booking with [a named human]." Generic "Schedule a meeting with the team" copy collapses the rep card into a calendar embed and loses the pre-frame. For a Terraplex dealer this means [Sam Welton](../../../content/compliance-program.md) on a compliance-consultation page, the dealer principal on a demo-flight page, the service manager on a fleet-check-in page — one named person per booking type.
- Don't combine this pattern with the [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md) on the same page. Both occupy the "conversion moment" slot; the calendar is for "I'm ready to talk now," the topic-checkboxes form is for "I want to talk about specific things." Pick one per page.
- Don't combine with the [dealer-locator map](./refero-71707203-dealer-locator-map.md) on the same page either — calendar booking assumes the customer already knows which dealer / location they want; the locator is for customers who don't. Use the locator first, then link to per-dealer booking pages.
- Don't add the calendar widget if the dealer doesn't actually keep its calendar in sync with the embed (Calendly, Cal.com, Savvy, internal CRM). A booking page that returns "we'll get back to you to confirm" defeats the affordance — the customer already filled out a form, the calendar was decoration, and the dealer's real intake is human-mediated. If the dealer can't or won't sync, drop this pattern and use [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md) instead.
- Don't pre-fill the form fields with "Refero" or sample placeholder copy for the sake of the screenshot. In production, fields are empty with helpful placeholders ("name@company.com", "+1 555…").
- Don't make the form longer than ~7 fields. The whole reason the calendar is visible is that the form is short. A 12-field intake form pushes the calendar below the fold, and the affordance dies — the page becomes a regular contact form with a calendar afterthought.
- Don't surface a price or quote inside the calendar widget — the widget books a meeting; the meeting is where price is discussed. Mixing pricing into the booking widget collapses the conversational register into a quote calculator (which violates [content/pricing-policy.md](../../../content/pricing-policy.md) MAP rules anyway).

## Adaptation notes (cross-archetype)
- CLEAN: native fit. White card on off-white page, brand-navy selected-timeslot fill, brand-navy "Schedule" pill. Rep avatar in a clean circle frame, role/title in muted gray. Form fields with light-gray inputs and a thin charcoal focus ring. See [CLEAN archetype](../../../archetypes/clean.md). Pairs with the [centered-screenshot-trio hero](../../hero/research/refero-f619e700-centered-screenshot-trio.md).
- FIELD: cream background (~#F4EFE7), warm-cream card with a soft warm drop shadow. Rep avatar is a real farmer-dealer or service operator — golden-hour lighting, no studio chrome. Selected timeslot fills with deep amber; "Schedule" pill in amber with cream label. Form input chrome uses torn-cream-paper inset rather than hard borders. See [FIELD archetype](../../../archetypes/field.md). Pairs with the [manifesto-creed hero](../../hero/research/refero-b86571f2-manifesto-creed-hero.md) on a parent landing page.
- TERRAIN: the calendar widget IS the page's main interactive band — the rest of the page is empty. Add a thin topographic divider at the top and bottom of the card (the card sits in the page's "light band" register; the dividers signal the alternating rhythm even though no second band actually shows up). Selected-timeslot fill in clay or sage. See [TERRAIN archetype](../../../archetypes/terrain.md).
- FORGE: invert to a dark variant — near-black card (~#0E0E0E) on a charcoal page (~#1A1A1A), white text and form labels, ember-orange selected-timeslot fill and ember "Schedule" pill. Rep avatar gets a thin ember rim-light treatment. Form input chrome is a slightly-lighter dark inset (~#222) with no visible border. See [FORGE archetype](../../../archetypes/forge.md). Pairs with the [hardware-lineup dark hero](../../hero/research/refero-0945a36c-hardware-lineup-dark-hero.md) on a parent product page.
- TACTICAL: tighten geometry — sharper rectangle card (~6px corner radius instead of pill), precision-sans labels in tracked all-caps for field labels ("EMAIL / COMPANY / WEBSITE"), monochrome white-on-near-black with a single steel-cyan selected-timeslot fill and "Schedule" rectangle CTA. Add a thin 1px corner-bracket detail to the card corners (consistent with the archetype's bracket motif). The chat-bubble support widget gets the same bracket treatment. See [TACTICAL archetype](../../../archetypes/tactical.md). Pairs with the [topographic layered-bg hero](../../hero/research/refero-ae84b62a-topographic-layered-bg.md).
- PATRIOT: cream background, cream card with a soft warm drop shadow. Rep avatar of the named dealer principal or [Sam Welton](../../../content/compliance-program.md) in late-afternoon light. Selected-timeslot fill in deep navy; "Schedule" pill in deep navy with cream label. The single barn-red moment is the small "required field" asterisk (a tiny red star, the literal flag color at the smallest scale on the page). Don't add stripe rules or stars to the card — the rep card pre-frame already carries the heritage register. See [PATRIOT archetype](../../../archetypes/patriot.md). Pairs with the [manifesto-creed hero](../../hero/research/refero-b86571f2-manifesto-creed-hero.md).

## Source brief
Marathon's [/demo](https://onmarathon.com/demo/) page — a SaaS booking interface using a Calendly-style embed with a rep-card pre-frame. The library-worthy move is the structural cadence (left text column with 2-line heading + minimal nav → right elevated card stack with rep card → form → calendar grid → timeslot picker → single Schedule CTA), reusable for a Terraplex dealer offering scheduled demo flights, [Part 107 prep consultations](../../../content/compliance-program.md), [Sam Welton compliance reviews](../../../content/compliance-program.md), or annual fleet check-ins. The rep-card pre-frame is the credibility move that differentiates this pattern from a bare scheduling embed — the customer is booking a meeting with a person, not with a calendar.

Sibling contact notes (four structurally distinct patterns total now):
- [two-card decision pivot](./refero-376ca585-two-card-decision-pivot.md) — intent-route first, form later (sales vs support split)
- [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md) — form-led with conversation-prep checkbox preview
- [dealer-locator map](./refero-71707203-dealer-locator-map.md) — full-screen map with location-type filter chips for ~10+ location networks
- this note — calendar-led booking with named-rep pre-frame for "ready to talk now"

Pick guide: locator first if the customer doesn't know which dealer; pivot if they need to choose between sales/support; checkboxes-form if they need to articulate what they want to discuss; this calendar pattern if they just want a confirmed time with a named person.
