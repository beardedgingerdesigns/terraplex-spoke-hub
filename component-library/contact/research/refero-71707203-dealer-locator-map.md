## Why this is library-worthy
A full-screen continental map as the primary contact-page surface, overlaid with a stacked column of location-type filter chips ("Stores", "Service", "Self-Serve Demo Drive", "Body Shops", "Superchargers", "Delivery Centers") and a single search field — the page IS the locator, not a contact form. Solves a use case the existing contact patterns ([two-card decision pivot](./refero-376ca585-two-card-decision-pivot.md), [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md)) don't address: a multi-location dealer network where the customer's first question is "which one is closest, and what does it do?" Directly relevant for a Terraplex hub-served dealer ecosystem that may eventually surface the [49-location dealer network](../../../research/manufacturers/terraplex.md) as a single find-a-dealer page across the platform; also useful for any single-dealer site with multiple locations (sales / service / training / Part 137 ops) that benefit from being filtered by service type.

## What to take
- The page is one full-bleed interactive map (continental US for a national network; state or county-level for a regional dealer) — the map fills 100% of the viewport below the site nav. No hero band, no marketing copy above; the map IS the page.
- A single floating filter-chip column anchors to the upper-left of the map (~24px inset from viewport edges). Chips are stacked vertically (one per line, not horizontal pills), each ~36–40px tall, white background with a 1px hairline border, ~16–20px corner radius, and a small monochrome circle-icon on the left followed by the label. Chips toggle visibility of one location-type layer on the map; multiple chips can be active simultaneously.
- Above the chip column, a search field pinned at the very top of the floating toolbar: ~280–360px wide, white, rounded, single placeholder ("Enter location"). Submitting jumps the map to that location and re-clusters pins.
- Map pins are color-coded by location type (matching the chip icon color) at small scale; at higher zoom they expand to per-pin info popovers with address, hours, phone, and a "Get Directions" link. Pin density is the credibility move — a sparsely-populated map for a 49-location dealer network reads weaker than a single dense map of one state with 8 service centers.
- Map controls (zoom +/–) sit at the right edge mid-height; a small chat-bubble support widget pins to the bottom-right corner. No other chrome competes for attention.
- The whole page is monochrome (light-gray map base, charcoal text, neutral hairline borders) except for pin colors. Restraint is the affordance — the map data is the visual treatment.

## What NOT to take
- Don't import Tesla's specific location-type vocabulary ("Superchargers", "Self-Serve Demo Drive", "Delivery Centers") — those are EV-specific. For a Terraplex dealer the chip column is "Sales / Demos", "Service Center", "Operator Training", "Part 137 Operations", "Compliance / Sam Welton" — pick the 3–6 location types the dealer network actually has, no more.
- Don't add this pattern if the dealer has only one location. The whole affordance depends on multiple pins; a single-location map with one pin reads as a contact-info embed and doesn't earn the full-screen treatment. Use [two-card decision pivot](./refero-376ca585-two-card-decision-pivot.md) or [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md) instead.
- Don't combine this pattern with a contact form on the same page. The locator IS the conversion moment — the customer who finds the right pin clicks "Get Directions" or calls the listed number. A form below the map dilutes the page's job and competes with the popover CTAs.
- Don't import the live-map embed if the dealer network is small enough to fit in a flat list. ~5 locations or fewer reads better as a typography grid (see [two-card decision pivot's offices grid](./refero-376ca585-two-card-decision-pivot.md)). The map earns its weight only at ~10+ locations.
- Don't use the floating-toolbar pattern with a pinned-bottom mobile view that hides the map. On mobile, the chip column collapses into a horizontal scroll strip below a sticky search field; the map remains primary. Hiding the map on mobile collapses the pattern into a generic location list and loses the cross-network legibility move.
- Don't fabricate pin density. If the dealer's actual network is sparse, zoom to a tighter region (state / county) so the map looks populated rather than empty. An empty continental US is worse than a cropped tight regional view with 6 pins.
- Don't add a CTA inside the map (e.g. a "Schedule Demo" floating button). The pattern's restraint depends on the chip filters being the only interactive overlay; a marketing CTA fights the locator job.

## Adaptation notes (cross-archetype)
- CLEAN: native fit. White map base (Mapbox or Google Maps default light style), charcoal pin clusters with a single brand-navy active state, hairline borders on the floating chip column. Search field uses the brand's input chrome. See [CLEAN archetype](../../../archetypes/clean.md).
- FIELD: warm-cream-tinted map style (custom Mapbox style with cream landmask, soft-amber roads), warm-charcoal chip text, single deeper-amber accent on the active filter and pin color. Pin icons are warm-charcoal stroked. The search field carries the same cream-paper inset as the rest of FIELD's form chrome. The pattern's monochrome restraint pairs well with FIELD's no-motion rule — the map is interactive but not animated. See [FIELD archetype](../../../archetypes/field.md).
- TERRAIN: this slots in as a long sustained "light band" — the map base reads as the topographic backdrop. Use a custom Mapbox style with contour lines visible at higher zoom (TERRAIN's signature topographic motif); chip column accent is clay or sage. Pin color picks up the archetype's clay accent. The bottom of the page (below the map) can transition to a dark band with a topographic divider for any closing copy or CTA. See [TERRAIN archetype](../../../archetypes/terrain.md). Sibling: [topographic layered-bg hero](../../hero/research/refero-ae84b62a-topographic-layered-bg.md) which uses the same contour-line vocabulary at hero scale.
- FORGE: dark map base (Mapbox dark style or Google Maps dark mode) with charcoal-to-near-black landmask and muted-orange road network. Floating chip column is a near-black surface with white text and a single ember-orange accent on the active chip. Pin clusters are ember-orange at small scale, expanding to dark-card popovers with white text. Map controls lift slightly off the dark surface (~#1F1F1F card with a thin charcoal hairline). Pairs naturally with the [hardware-lineup dark hero](../../hero/research/refero-0945a36c-hardware-lineup-dark-hero.md) above the map (introduces the lineup, then the locator finds where to demo it) and the [dark feature bands product-block](../../product-block/research/refero-eb12db31-dark-feature-bands.md) on adjacent product pages. See [FORGE archetype](../../../archetypes/forge.md).
- TACTICAL: dark map base with tighter geometry — sharper rectangle filter chips (~6px corner radius instead of pill), precision-sans labels in tracked all-caps ("STORES", "SERVICE", "TRAINING"), monochrome white-on-near-black with a single steel-cyan accent on the active chip and pin color. Add a thin 1px corner-bracket detail to the chip column container (consistent with the archetype's bracket motif). Map pins are simple cyan dots, not pins, so the data overlay reads as a precision tracking display rather than a marketing locator. See [TACTICAL archetype](../../../archetypes/tactical.md). Pairs with the [hardware-lineup dark hero](../../hero/research/refero-0945a36c-hardware-lineup-dark-hero.md) for a TACTICAL page rhythm.
- PATRIOT: cream-tinted map base (warm off-white landmask, muted-gray roads — warmth comes from the basemap, not from accent saturation), deep-navy chip text on cream chip surfaces. Pin color is deep navy; active filter is barn-red — the only saturated moment on the page, which keeps the flag colors at micro-scale. Do NOT add flag-stripe rules or star motifs around the floating toolbar; the locator's restraint is what keeps the pattern from reading as kitsch. The map covers the network across the dealer's territory (state-level for most, regional for multi-state dealers); for the Terraplex 49-location network this naturally pulls toward the warm-American-farmland register without needing chrome. See [PATRIOT archetype](../../../archetypes/patriot.md). Pairs with the [manifesto-creed hero](../../hero/research/refero-b86571f2-manifesto-creed-hero.md) on adjacent pages.

## Source brief
Tesla's [/findus](https://www.tesla.com/findus) page — the global locator across stores, service centers, body shops, superchargers, demo drives, and delivery centers. The library-worthy move is the structural cadence (full-bleed interactive map → floating filter-chip column with stacked location-type toggles → single search field → minimal map controls), reusable for a Terraplex dealer's multi-location network or for the broader [49-location Terraplex dealer network](../../../research/manufacturers/terraplex.md) at the hub level. The pattern is most directly transferable because Tesla, like Terraplex, sells equipment-plus-service through a network where the customer's first question is "where" and "what kind" — not "tell me more."

Sibling contact notes: [two-card decision pivot](./refero-376ca585-two-card-decision-pivot.md) for a 1–2-location dealer with sales-vs-support intent routing; [form-with-topic-checkboxes](./refero-23938dfd-form-with-topic-checkboxes.md) for any dealer where the contact page's job is conversation-prep rather than location-discovery. This dealer-locator pattern is the right pick when the network's breadth is the credibility move.
